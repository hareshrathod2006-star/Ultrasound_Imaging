🧠 Start Here: What Problem Are We Solving?

You already know that ultrasound works by sending sound waves into the body and listening to the echoes. But here's the key question: how do you turn those raw echoes into a picture?

The answer is beamforming — and DAS (Delay-And-Sum) is the most fundamental way to do it.

🎯 The Core Idea of DAS — An Analogy

Imagine you're blindfolded in a large room, and you clap your hands once. Your two ears hear the echo come back. Because your ears are in slightly different positions, the echo from the left wall reaches your left ear slightly before your right ear.

By measuring WHICH ear heard it first and HOW MUCH earlier, you can figure out WHERE that wall is.

DAS does exactly this — but with 128+ transducer elements instead of 2 ears, and with millions of pixels instead of just one wall.

📐 DAS in 3 Simple Steps

1

DELAY — Calculate travel time to every pixel

For each pixel (x, z) in the image grid, calculate exactly how long it would take sound to travel FROM the transducer TO that pixel AND BACK to each receiving element. This gives you a time delay for each element.

2

ALIGN — Shift the signals using those delays

Each element's recorded signal is shifted (delayed) so that echoes from the target pixel line up in time across all elements. This is called "rephasing."

3

SUM — Add all the aligned signals together

When you add all elements' aligned signals, echoes from the target pixel add up coherently (get louder), while noise and echoes from other locations cancel out. Result: one bright pixel value.

🔁 Repeat this for EVERY pixel in the image grid. That's your ultrasound image!

🔬 What is this PALA project specifically?

PALA = Performance Assessment of Localization Algorithms. It's a benchmark project from a French research lab (Sorbonne University) for Ultrasound Localization Microscopy (ULM) — a super-resolution technique that tracks microbubbles in blood vessels to image them 10x beyond normal ultrasound resolution. DAS beamforming is step one: turning raw RF data into images that you then analyze.

📡 Key Physical Concepts You Need

① RF Signal — What the transducer actually records

RF = Radio Frequency signal. It's the raw voltage vs. time waveform picked up by each transducer element after a sound pulse is sent. It oscillates at the transducer's center frequency (e.g., 15 MHz). Think of it as the "raw audio" — not yet converted into an image.

② IQ Signal — The demodulated version

IQ = In-phase / Quadrature. Instead of storing the fast-oscillating RF signal, we extract the "envelope" information as a complex number (I + jQ). This is mathematically equivalent but easier to work with. The magnitude |IQ| gives you the brightness of each pixel.

RF signal (15 MHz oscillating) → demodulate → IQ (complex, slower)
Like converting a radio carrier wave into just the audio information it carries

③ The Delay Formula — The heart of DAS

τ(x,z, element_e) = (d_tx + d_rx) / c
d_tx = distance from transmitter to pixel (x,z) | d_rx = distance from pixel to element e | c = speed of sound (~1540 m/s in tissue)

For a plane wave sent at angle θ:

d_tx = z·cos(θ) + x·sin(θ)
d_rx = √[(x - xe)² + z²]
xe = x-position of element e on the transducer array

④ Plane Wave Compounding — Why use multiple angles?

Instead of focusing on one beam, you fire a flat "plane wave" across the entire image at different steering angles (e.g., -10°, -5°, 0°, +5°, +10°). Each angle gives a low-quality image. You then coherently add (compound) them together.

✅ Result: much better image quality (contrast, resolution) while keeping high frame rate — critical for tracking fast-moving microbubbles!

In the code: PARAM.compound = 1 means "yes, sum all angles together"

⑤ F-number — Controlling which elements contribute

F-number = focal_depth / aperture_width. It controls which transducer elements are "allowed" to contribute to each pixel. Elements too far to the side are ignored (they'd add noise, not signal). In the code: PARAM.fnumber = 1.9

⑥ B-mode Image — What you finally see

B-mode [dB] = 20 · log₁₀( |IQ| )
Convert complex IQ magnitude to decibels for display. Bright = strong echo. Dark = weak echo.

In the code: Bmode = 20*log10(abs(IQ))

🗺️ Understanding the Pixel Grid

mesh_X, mesh_Z are 2D matrices where every entry gives the (x,z) coordinate of one output pixel. Created with MATLAB's meshgrid. The beamformer loops over every pixel to compute its intensity.
PARAM.z = (Origin_z + [0,1,2,...,Nz-1] × Δz) × wavelength
PARAM.x = (Origin_x + [0,1,2,...,Nx-1] × Δx) × wavelength
PDelta gives the pixel spacing in wavelengths; multiply by wavelength to get meters

🔧 BF_DelayAndSum — The Core Function Explained

This is where the actual DAS algorithm lives. Let's dissect every line.

INPUT
Function Signature
1function IQ = BF_DelayAndSum(SIG, PARAM, X, Z)
SIG: Cell array of IQ signals. SIG{1} = signal for angle 1, SIG{2} = angle 2... Each is [Nsamples × Nelements]
PARAM: The parameter structure (all the physics constants)
X, Z: 2D pixel coordinate grids from meshgrid — shape [Nz × Nx]
IQ (output): Beamformed complex image — shape [Nz × Nx]. Take abs() to get brightness.
INIT
Initialize Output
2IQ = zeros(size(X), class(SIG{1}));
3if ~isfield(PARAM, 'compound')
4 PARAM.compound = 1; % default to compounding
5end
Creates an output image filled with zeros, same data type as input signal (single-precision float for memory efficiency).
The isfield check is a safety net — if someone calls this function without setting compound, it defaults to ON.
LOOP
Main Angle Loop — The DAS Core
6for k = 1:length(PARAM.angles_list) % for each steering angle
7 PARAM.theta = PARAM.angles_list(k); % set current angle
8 if PARAM.compound
9 IQ = IQ + BF_das_rephaseSignal(SIG{k}, PARAM, X, Z); % ADD to same image
10 else
11 IQ(:,:,k) = BF_das_rephaseSignal(SIG{k}, PARAM, X, Z); % SEPARATE images
12 end
13end
For each angle k:
1. Set PARAM.theta = that angle
2. Call BF_das_rephaseSignal — this does the actual delay computation and summation for ONE angle. It returns a full 2D image.
3. If compounding: add the result to the accumulating IQ image (coherent summation)
4. If not compounding: store it as a separate "slice" — IQ(:,:,1), IQ(:,:,2), etc.

BF_das_rephaseSignal (not shown here) is where τ = (d_tx + d_rx)/c is computed for every pixel, the signal is interpolated at that time delay, and summed across all elements.

🔧 Using the Code on Your Own Data

⚠️ To use with YOUR OWN data, you need to fill PARAM manually. Here's the minimum you need:
% Minimum PARAM for your own data:
PARAM.f0 = 15e6; % your transducer center freq [Hz]
PARAM.fs = 60e6; % your sampling frequency [Hz]
PARAM.c = 1540; % speed of sound [m/s]
PARAM.xe = linspace(-0.02,0.02,128); % element positions [m]
PARAM.Nelements = 128;
PARAM.t0 = 0; % adjust if needed
PARAM.angles_list = 0; % single straight-ahead angle
PARAM.fnumber = 1.5;
PARAM.compound = 0; % no compounding (single angle)
PARAM.wavelength = PARAM.c/PARAM.f0;
